Model Support

⚠️ No model family is supported yet. Selecting the first one is Phase 0 of the roadmap.

Kipp will initially support one explicitly selected family and a small, documented set of checkpoints.

#What the support record will contain

  • Exact upstream model identifiers and revisions
  • Model, tokenizer, and weight licenses
  • Supported parameter counts and architectural variants
  • Accepted source and runtime weight formats
  • Quantization and numeric-format support by backend
  • Tokenizer and chat-template compatibility
  • Minimum memory and hardware requirements
  • Known correctness or quality limitations

#The bar for a new checkpoint

Supporting a new checkpoint will require:

  1. A reproducible conversion procedure (see tools/convert_to_gguf.py).
  2. Reference test vectors generated for the checkpoint.
  3. Validation on every backend claimed to support it, on real hardware.
Kipp is not intended to accept arbitrary GGUF files.
