Contributing

Kipp's contribution constraints live in AGENT.md and apply to every contributor — human or AI agent.

#Language rules

  • C11 is the core language; no compiler-specific extensions unless a backend genuinely requires them.
  • Objective-C only where Metal requires a bridge — confined to src/metal/.
  • Metal Shading Language for Apple GPU kernels; CUDA for NVIDIA kernels, isolated under src/cuda/.
  • Python and shell are tooling-only. No PyTorch, JAX, ONNX Runtime, or any generic tensor framework anywhere in src/.

#Engineering discipline

  • A change to one backend must never silently break another — keep the paths isolated.
  • Keep model loading mmap-backed; avoid eager full-weight copies into RAM.
  • Any change that could affect a backend must be tested on that backend's real hardware before being called done. If you can't test it, say so explicitly.
  • Every change to the KV cache, scheduler, or quantization path needs a corresponding test in tests/.
  • Prefer readability over cleverness; favor small, specific functions over generic abstractions.
  • The CPU backend exists for reference and debugging — never optimize it at the GPU paths' expense.
  • When in doubt between correctness and speed, choose correctness.

#Provenance

The reference codebases in context/inspiration-notes.md — antirez/ds4, llama.cpp, ggml, whisper.cpp, llm.c, nano-vllm, vLLM, SGLang — are for architecture study only.

Do not vendor or copy their code. Read them for ideas, reimplement cleanly, and record the repository, file, revision, and license of anything you adapt beyond a general idea.
